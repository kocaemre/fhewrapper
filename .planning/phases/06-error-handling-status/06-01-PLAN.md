## Phase Goal

**As a** judge or developer using the live dApp, **I want to** see every write flow's state and failures in clear, human-readable terms — with toasts, a stage view, and a real explorer link — **so that** I always know what is happening and never see a raw revert string or a silent hang.

<objective>
Systematize the error/status handling built ad-hoc across Phases 3-5 into ONE consistent typed model, and prove it end-to-end on the faucet flow (the flow with the biggest gap: no stage indicator, no toast, no explorer link today). Consolidates the four per-flow error maps behind a single `toAppError` classifier, adds the shared status primitives (themed toast helper + Sepolia explorer link), and wires them into the faucet claim.

Purpose: UX-01 (typed human-readable errors for every failure mode) + UX-02 (toasts / tx-stage / explorer links across write flows) are an explicitly-judged axis. This plan establishes the reusable system and delivers the first fully-treated write flow; Plan 06-02 applies the same system to wrap + unwrap.
Output: `lib/appError.ts` (+ test), `lib/explorer.ts` (+ test), the four flow maps refactored to delegate, `components/status/txToast.tsx`, `components/status/ExplorerTxLink.tsx`, `components/faucet/FaucetStageIndicator.tsx`, and a fully-wired faucet claim (stage + toast + explorer + typed error).

This is a SYSTEMATIZATION phase — NO new `@zama-fhe` SDK surface is added or changed. Every symbol referenced already ships in the installed EXACT `@zama-fhe/react-sdk` 3.0.0. No package installs.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| thrown error -> UI message | Wallet/SDK/wagmi errors (incl. raw revert strings) cross into user-visible copy |
| onchain token metadata -> toast/UI | Untrusted `symbol`/`name` strings from token contracts are interpolated into toast + row text |
| tx hash -> explorer href | A tx hash becomes the path of an outbound `sepolia.etherscan.io` link |

| T-06-02 | Tampering (XSS) | components/status/txToast.tsx, components/faucet/FaucetRow.tsx | medium | mitigate | All title/desc/token-symbol text passed as React children (react-hot-toast JSX escaping); never `dangerouslySetInnerHTML` (T-02-01 carried) |
