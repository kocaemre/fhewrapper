## Phase Goal

**As a** judge or developer using the live dApp, **I want to** see every write flow's state and failures in clear, human-readable terms — with toasts, a stage view, and a real explorer link — **so that** I always know what is happening and never see a raw revert string or a silent hang.

<objective>
Apply the unified error/status system from Plan 06-01 to the two remaining write flows (wrap + unwrap), and make the unwrap async wait visibly reassuring. Adds themed toasts + a working Sepolia explorer link to the wrap and unwrap flows, migrates their error rows to the shared `toAppError` chip+body model, and polishes the unwrap "pending decryption" wait into an explicit, reassuring state (UX-03).

Purpose: complete UX-02 (toasts / tx-stage / explorer links) across ALL write flows and deliver UX-03 (the async unwrap wait as production-grade UX). The wrap/unwrap 4-stage indicators and error maps already exist (Phases 4/5) — this plan layers the toast + explorer + reassurance on top and points the error rows at the unified model.
Output: `hooks/useWrap.ts` + `hooks/useUnwrap.ts` expose the submitted tx hash; `WrapPanel` + `UnwrapPanel` gain pending/success/error toasts, an inline explorer link, and a unified typed error row; `UnwrapStageIndicator` (and/or the panel) presents the decrypting/finalizing wait as reassuring copy.

This is a SYSTEMATIZATION phase — NO new `@zama-fhe` SDK surface. Every SDK symbol already ships in the installed EXACT 3.0.0. No package installs. The two-tx SDK/wallet/relayer path stays owned by the SDK and is proven live in 06-UAT.
</objective>
