## Phase Goal

**As a** judge or developer evaluating the app, **I want to** decrypt my confidential (ERC-7984) balance for any token via a correct one-signature EIP-712 flow, **so that** I can see and trust the real cleartext result of every wrap and unwrap.

<objective>
Deliver the paste-an-address decrypt slice (DEC-02): a dedicated /decrypt screen where a connected Sepolia wallet pastes ANY ERC-7984 address — inside OR outside the registry — validates it, signs one reusable EIP-712 permit, and reveals its confidential balance with the blur→reveal treatment. Reuses the 03-01 shared engine + pure-logic, so a permit granted on a registry card also unlocks the panel (DEC-03).

Purpose: The "any token" claim (paste-an-address) is a distinct judging axis proving the single non-token-specific permit works for arbitrary confidential tokens.
Output: A tested address-validation helper, three decrypt UI atoms, the composed DecryptPanel, and the /decrypt route under ChainGuard.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user paste → app | Arbitrary user-supplied string interpreted as a contract address (untrusted). |
| client → wallet (MetaMask) | EIP-712 permit signature for the pasted token. |
| client → Zama relayer/gateway | Decrypt of the pasted token's on-chain handle. |
