## Phase Goal

**As a** judge or developer evaluating the app, **I want to** decrypt my confidential (ERC-7984) balance for any token via a correct one-signature EIP-712 flow, **so that** I can see and trust the real cleartext result of every wrap and unwrap.

<objective>
Deliver the first real end-to-end decrypt slice: a connected Sepolia wallet clicks Decrypt on a registry PairCard and sees the correct cleartext balance (DEC-01), authorized by one reusable EIP-712 permit (DEC-03), with graceful typed failure and zero-balance handling (DEC-04). This plan builds the SHARED decrypt engine (`useUserDecrypt`) + tested pure-logic helpers (error taxonomy, decimals formatting) that Phase 3's paste panel and Phases 4/5 (wrap/unwrap) reuse.

Purpose: The wrapped balance is an opaque handle without decryption — this slice is the tool that lets a judge SEE every other flow's result. Correctness of the EIP-712 user-decryption is a top judging axis.
Output: Two test-locked pure functions, the reusable `useUserDecrypt` hook, and per-card blur→reveal decrypt wired into the existing Phase-2 PairCard.
</objective>
